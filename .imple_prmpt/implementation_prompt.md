# 股票自動篩選與通知系統實作提示 (Implementation Prompt)

這是一個用於開發「每日股票自動篩選與信件通知系統」的實作提示 (Prompt)。請將這段提示提供給 AI 或開發人員，以生成完整的程式腳本（建議使用 Python）。

---

## 系統核心需求

1. **自動排程執行**：系統需排程於 **每個交易日下午 1:00 整** 自動觸發執行。
2. **自動通知**：程式完成資料抓取與篩選後，需將符合條件的股票清單與相關數據自動發送到指定的 Email 信箱。

## 股票篩選條件 (於下午 1:00 判斷)

請設計資料獲取邏輯，串接即時報價 API 與歷史資料庫，並嚴格按照以下條件進行篩選每支股票：

1. **即時漲幅**：當下（1:00）的股價漲幅介於 **3% ~ 5%** 之間。
2. **成交強度量比**：當下的預估量或累積量比（相較於前 5 日平均成交量） **> 1**。
3. **換手率**：當日的預估/實際換手率介於 **5% ~ 10%** 之間。
4. **市值規模**：個股總市值介於 **200 億 ~ 1000 億台幣** 之間。
5. **近期強勢紀錄**：近 **20 個交易日內**，至少曾有一次**漲停**紀錄。
6. **當日走勢型態 (分時均線)**：
   - **均線之上**：開盤至 13:00 的股價走勢，絕大多數時間皆維持在**分時均線 (VWAP - 成交量加權平均價) 上方**。
   - **回踩有撐**：需具備「偶爾回踩均線附近時，立刻有承接買盤反彈」的走勢特徵（演算法可設計為：當股價距離 VWAP 小於 0.5% 時，後續 1~3 盤內價格明顯回升，且未有實質跌破均線的狀況）。
7. 上述所有數值都要可以調整，因此要寫一個好理解的UI讓使用者調整參數

## 技術架構與實作建議

* **程式語言推薦**：`Python`
* **資料來源建議**：
  * **即時報價與分時均價**：使用fubon neo api
  * **歷史資料 (20日漲停紀錄、市值、均量)**：可使用 `FinMind`、`yfinance`、或爬取台灣證券交易所/櫃買中心每日資料。
* **排程機制**：
  * 若在本地端執行：使用 Windows `工作排程器 (Task Scheduler)`，或直接使用 Python `schedule` / `APScheduler` 套件常駐運行。
* **Email 發信模組**：
  * 使用 Python 內建的 `smtplib` 與 `email.mime` 模組。
  * 建議將 Email 帳號與應用程式密碼 (App Password) 儲存於 `.env` 環境變數檔中以策安全。
* **例外處理**：
  * 需判斷當日是否為「台股交易日」，若為假日則不執行篩選。
  * 加上 API 請求失敗的重試機制與錯誤紀錄 (Logging)。
